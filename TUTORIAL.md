# 汽水音乐 Cover 上传过审全攻略（2026实测版）

> 从 Suno 生成到汽水音乐上架，全流程避坑指南

---

## 一、先搞清底层逻辑

汽水音乐的审核机制和你想象的不一样：

| 你以为 | 实际情况 |
|--------|----------|
| 会严格查版权 | **优先级：音频质量 > 版权声明** |
| 翻唱必须标注 | **标 Cover 反而容易被退回** |
| AI 标注会限流 | **目前不查 AI 来源** |
| 需要原版权授权 | **小量上传根本没人管** |

**核心策略：把 Cover 当成「原创」上传。**

---

## 二、Suno 生成阶段

### 2.1 Prompt 怎么写

不要写「翻唱周杰伦的晴天」，写：

```
Mandarin pop ballad, piano-driven, melancholic male vocal, 
80-100 BPM, emotional storytelling, clear articulation
```

用 [Suno Funnel](https://github.com/2569658930/tan) 自动生成：
```bash
python3 suno_funnel.py full 周杰伦 "雨天下的便利店" ballad
```

### 2.2 生成技巧

- **Custom Mode** 开，手动输入歌词
- 歌词用 `[Verse]` `[Chorus]` `[Bridge]` 标记结构
- **不要**在 Suno prompt 里写原曲名、原唱名
- 一次生成 2-3 版，选音质最好的

### 2.3 BPM 避雷

Cover 的最大坑：**BPM 和原曲差太多听起来不像。**
- 原曲 90 BPM → Suno 出了 120 BPM → 完全不是那味
- 解决：Suno Funnel 的 `full` 命令自动输出参考 BPM

---

## 三、汽水音乐上传阶段

### 3.1 上传入口

`抖音音乐开放平台` → `汽水音乐人` → `音频投稿`

不是「翻唱投稿」！选**音频投稿**。

### 3.2 填写攻略

| 字段 | 填什么 | 为什么 |
|------|--------|--------|
| **分类** | 选「**原创**」 | 选翻唱需要版权证明 |
| **标题** | `{歌名} (Piano Cover)` | 不加「Cover」容易过，加了声明立场 |
| **标签** | `#原创 #AI创作 #钢琴` | 3-5个中性标签 |
| **描述** | `AI音乐创作实验作品` | 不写翻唱、不写原曲信息 |
| **封面** | 自己做的 800x800 图 | 别用原曲封面 |

### 3.3 常见的退回原因

1. **音频有杂音/断音** → 重生成，选 2 分钟以上的版本
2. **标题包含原唱名** → 删掉原唱名
3. **选了「翻唱」分类** → 改成原创
4. **音频太短** → 至少 1:30 以上
5. **封面违规** → 别有露骨的/带其他平台水印的

---

## 四、Remix 过审特别篇

Remix 比 Cover 更容易过，因为：

- 没有「原曲」概念 → 天然是原创
- 电子/舞曲风格 → 审核偏重节奏和质量

Remix 快捷生成：
```bash
python3 suno_funnel.py remix "深夜城市霓虹" electronic
```

**标题技巧：**
- ❌ `晴天 EDM Remix`
- ✅ `夏日晴空 (Electronic Version)`

## 五、收益路径

| 路径 | 门槛 | 单曲收益 |
|------|------|----------|
| 播放分成 | 0粉 | 极低 |
| 抖音热歌计划 | 需一定播放量 | 几百到几千 |
| 接单做 Cover | 展示 5+ 作品 | 几十到几百/单 |
| 卖 Suno prompt | 建工具/GitHub | 打赏/Sponsor |

---

## 六、踩过的坑

1. **用 Suno 生成的歌手动改 meta data → 被退回**  
   → 别改，直接上传

2. **一天传太多 → 限流**  
   → 控制在 2-3 首/天

3. **Cover 标题写了原曲名 → 被版权相关退回**  
   → 标题完全不要出现原曲信息

4. **用原曲封面 → 图片审核不通过**  
   → 自己做一个简单封面
